## Q1: 可达性分析详解

### 1.1 支撑向量方向选择

**问题**: 支撑向量方向是如何选择的？

**答案**: 8 方向均匀分布是工程实践中的最佳权衡。

**方向选择原理**:

```
方向数    优点                    缺点
────────────────────────────────────────────
4 方向    计算最快                过于保守，可达集近似差
8 方向    计算快 + 近似好 ✅       轻微保守
16 方向   近似更好                计算量翻倍
32 方向   非常精确                计算量大，收益递减
∞方向     精确可达集              计算不可行
```

**8 方向具体角度**:
```
索引  角度    方向向量      物理含义
─────────────────────────────────────────
0     0°      [1, 0]        前方
1     45°     [√2/2, √2/2]  右前
2     90°     [0, 1]        右方
3     135°    [-√2/2, √2/2] 右后
4     180°    [-1, 0]       后方
5     225°    [-√2/2, -√2/2]左后
6     270°    [0, -1]       左方
7     315°    [√2/2, -√2/2] 左前
```

**为什么选择 8 方向**:
1. **凸多边形闭合**: 8 个半平面交集形成闭合凸多边形
2. **计算效率**: 8 维输出，GRU 推理 <0.05ms
3. **保守性适中**: 与真实可达集误差 <10%
4. **碰撞检测简单**: 多边形 - 圆相交检测 O(8)

### 1.2 为什么输出只有位置，没有速度？

**问题**: 支撑函数只表示位置边界，那速度信息呢？

**答案**: 速度信息隐含在状态序列中，支撑函数是"可达位置集"的边界。

**详细解释**:

可达集定义:
$$\mathcal{R}(t+H) = \{x(t+H) : x(t) \in \mathcal{X}_t, u(\cdot) \in \mathcal{U}, d(\cdot) \in \mathcal{D}\}$$

其中 $x = [x, y, \theta, v, \omega]^T$ 是 5 维状态。

**但我们只关心位置可达集**:
$$\mathcal{R}_{pos}(t+H) = \{(x, y) : [x, y, \theta, v, \omega]^T \in \mathcal{R}(t+H)\}$$

**原因**:
1. **碰撞检测只需要位置**: 障碍物在 (x,y) 空间，不在速度空间
2. **可视化友好**: 2D 平面可绘制，5D 空间无法可视化
3. **计算简化**: 2D 支撑函数 (8 维) vs 5D 支撑函数 (需更多方向)

**速度信息的使用**:
- 速度包含在 GRU 输入中 (过去 10 帧的 v, ω)
- GRU 内部状态编码了速度信息
- 速度影响未来位置，间接影响可达集

### 1.3 底盘和机械臂是分开预测的吗？

**答案**: 是的，完全分开。

**原因**:

| 特性 | 底盘 GRU | 机械臂 GRU |
|------|----------|------------|
| **状态空间** | 5 维 [x,y,θ,v,ω] | 14 维 [q1-7, dq1-7] |
| **可达集维度** | 2D 平面 (x,y) | 3D 空间 (x,y,z) |
| **支撑方向** | 8 方向 (平面) | 6 方向 (±x,±y,±z) |
| **预测时域** | 1.0 秒 | 0.5 秒 |
| **碰撞对象** | 地面障碍物 | 桌面/天花板 |
| **训练数据** | 80,000 轨迹 | 50,000 轨迹 |

**分开预测的优势**:
1. **模型简化**: 两个小网络 vs 一个大网络
2. **训练高效**: 独立训练，互不干扰
3. **推理并行**: 可同时计算
4. **可解释性强**: 各自物理意义清晰

**耦合处理**:
- 机械臂运动影响整车重心 → Region 1 ZMP 检查
- 底盘加速影响机械臂基座 → Region 2 分别预测

### 1.4 可达集具体计算流程

**完整流程**:

```
步骤 1: 收集历史状态 (10 帧，0.2 秒)
───────────────────────────────────────
输入序列：[x(t-9), x(t-8), ..., x(t)]
每个 x = [x, y, θ, v, ω]

例如:
x(t-9) = [1.2, 0.5, 0.1, 0.3, 0.1]
x(t-8) = [1.3, 0.6, 0.15, 0.35, 0.12]
...
x(t)   = [2.5, 1.8, 0.5, 0.5, 0.2]


步骤 2: GRU 前向传播
───────────────────────────────────────
GRU 层 1: h₁ = GRU(x(t-9:t), h₀)  → 64 维隐藏状态
GRU 层 2: h₂ = GRU(h₁, h'₀)       → 64 维隐藏状态
全连接 1:  z = ReLU(W₁·h₂ + b₁)   → 32 维
全连接 2:  h = W₂·z + b₂          → 8 维输出

输出：h = [h₀, h₁, h₂, h₃, h₄, h₅, h₆, h₇]
      = [1.2, 0.9, 0.6, 0.8, 1.1, 0.7, 0.5, 0.8]


步骤 3: 从支撑值重构可达多边形
───────────────────────────────────────
每个支撑值 hᵢ 表示：在方向 dᵢ 上的最大延伸距离

支撑线方程：dᵢ·x ≤ hᵢ

8 条支撑线的交集 = 凸多边形

计算顶点 (相邻支撑线交点):
v₀ = intersect(line₀, line₁)
v₁ = intersect(line₁, line₂)
...
v₇ = intersect(line₇, line₀)

多边形顶点：[v₀, v₁, v₂, v₃, v₄, v₅, v₆, v₇]


步骤 4: 碰撞检测
───────────────────────────────────────
对于每个障碍物 (圆心 c, 半径 r):
  对于多边形每条边 (vᵢ, vᵢ₊₁):
    计算边到圆心的距离 d
    如果 d < r: 碰撞预警！

或者使用分离轴定理 (SAT):
  如果存在分离轴 → 无碰撞
  否则 → 碰撞


步骤 5: 计算碰撞时间 (TTC)
───────────────────────────────────────
TTC = (障碍物距离 - 安全裕度) / 相对速度

例如:
  障碍物距离 = 3.0m
  当前速度 = 0.5m/s
  TTC = (3.0 - 0.5) / 0.5 = 5.0 秒

如果 TTC < 2.0 秒 → 立即预警
```

---

**数学推导**:

**支撑函数定义**:
$$h_{\mathcal{K}}(d) = \max_{x \in \mathcal{K}} d^T x$$

**从支撑值重构多边形**:

给定 8 个方向 $d_i = [\cos\theta_i, \sin\theta_i]$ 和支撑值 $h_i$，多边形顶点为相邻支撑线的交点：

$$\text{line}_i: d_i \cdot x = h_i$$

$$\text{line}_{i+1}: d_{i+1} \cdot x = h_{i+1}$$

交点公式:
$$v_i = \text{solve}\begin{pmatrix} d_i^T \\ d_{i+1}^T \end{pmatrix} x = \begin{pmatrix} h_i \\ h_{i+1} \end{pmatrix}$$

$$v_i = \begin{pmatrix} \cos\theta_i & \sin\theta_i \\ \cos\theta_{i+1} & \sin\theta_{i+1} \end{pmatrix}^{-1} \begin{pmatrix} h_i \\ h_{i+1} \end{pmatrix}$$

## Q2: 被测对象训练状态

### 2.1 VLA 模型训练状态

**问题**: 被测对象 (OpenVLA) 是训练好的还是自己训练的？

**答案**: **简化仿真实现，未使用真实 OpenVLA 模型**。

**详细说明**:

| 组件 | 真实 OpenVLA | 本实验实现 |
|------|-------------|------------|
| Vision Encoder | ViT-B/16 (预训练) | 512 维随机特征 + 噪声 |
| Language Encoder | LLaMA-7B (预训练) | 简化为固定指令编码 |
| Action Decoder | Diffusion Transformer | 简化 DRL 策略 (随机 + 目标导向) |
| 训练数据 | 100 万轨迹 (Open X-Embodiment) | 无需训练，在线生成 |
| 推理延迟 | ~70ms @ 15Hz | ~0ms (简化) |

**为什么简化**:
1. **计算资源**: 真实 OpenVLA 需 GPU，推理慢
2. **实验重点**: 验证 RTA 有效性，而非 VLA 性能
3. **可重复性**: 简化实现易于复现
4. **3120 次试验**: 真实模型需 3000+ GPU 小时

**简化策略的合理性**:
- VLA 输出包含正常动作 + 故障时错误动作
- 故障注入机制模拟真实 VLA 失效模式
- RTA 系统不依赖 VLA 内部结构 (黑盒测试)
