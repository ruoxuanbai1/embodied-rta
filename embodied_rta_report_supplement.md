## 附录 C: 成功率计算方法

### C.1 成功判据

试验成功需**同时满足**以下条件：

```python
def is_success(info):
    """
    成功判据
    
    参数:
        info: dict, 环境步进返回的信息字典
    
    返回:
        success: bool, 是否成功
    """
    # 1. 无碰撞
    if info.get('collision', False):
        return False
    
    # 2. 无约束违反
    if info.get('violations', False):
        return False
    
    # 3. ZMP 稳定
    if not info.get('zmp_stable', True):
        return False
    
    # 4. 到达目标 (位置误差 <0.5m)
    target_x = 10.0
    current_x = info.get('base_x', 0)
    if abs(target_x - current_x) > 0.5:
        return False
    
    return True
```

---

### C.2 成功率统计

**单次场景 - 方法组合**:
$$SR_{m,s} = \frac{N_{success}(method=m, scenario=s)}{N_{total}}$$

其中 $N_{total} = 30$ (蒙特卡洛重复次数)

**方法平均成功率**:
$$\overline{SR}_m = \frac{1}{8}\sum_{s=1}^{8} SR_{m,s}$$

**场景平均成功率**:
$$\overline{SR}_s = \frac{1}{13}\sum_{m=1}^{13} SR_{m,s}$$

---

### C.3 置信区间计算

**95% 置信区间** (Wilson 分数区间):

$$\hat{p} = \frac{N_{success}}{N_{total}}$$

$$CI_{95\%} = \frac{\hat{p} + \frac{z^2}{2n} \pm z\sqrt{\frac{\hat{p}(1-\hat{p})}{n} + \frac{z^2}{4n^2}}}{1 + \frac{z^2}{n}}$$

其中 $z = 1.96$ (95% 置信水平), $n = 30$

**示例**: Ours_Full 在 S1 场景 30/30 成功
- $\hat{p} = 1.0$
- $CI_{95\%} = [0.88, 1.00]$

## 附录 D: 对比方法原理详解

### D.1 DeepReach (神经可达性分析)

**参考**: Bansal et al., ICRA 2021

**核心思想**:
使用神经网络求解 Hamilton-Jacobi-Issacs (HJI) 偏微分方程，计算高维状态空间的可达集边界。

**HJI 方程**:
$$\frac{\partial V}{\partial t} + \min_{u \in \mathcal{U}} \max_{d \in \mathcal{D}} \nabla V \cdot f(x, u, d) = 0$$

其中:
- $V(x,t)$: 价值函数 (可达集指示函数)
- $u$: 控制输入
- $d$: 扰动/不确定性
- $f(x,u,d)$: 系统动力学

**网络架构**:
```
Input (state x) → MLP (256→128→64) → Output (V value)
```

**训练方式**:
- 离线训练：100 万状态 - 时间样本
- 损失函数：HJI PDE 残差 + 边界条件
- 训练时长：12 小时 @ 8×V100

**优点**:
- 理论保证强 (形式化验证)
- 可处理 10+ 维状态空间

**缺点**:
- 离线训练时间长
- 泛化能力有限 (需重新训练适应新场景)

---

### D.2 Recovery RL (安全恢复强化学习)

**参考**: Thananjeyan et al., RA-L 2021

**核心思想**:
双策略架构 - 任务策略 (task policy) + 恢复策略 (recovery policy)，基于风险价值函数切换。

**风险价值函数**:
$$Q_{risk}(s) = \mathbb{E}_{\pi_{task}}\left[\sum_{t=0}^{\infty} \gamma^t c(s_t) | s_0 = s\right]$$

其中 $c(s) = 1$ 如果状态 $s$ 危险，否则 $0$。

**策略切换**:
$$\pi(s) = \begin{cases}
\pi_{task}(s) & \text{if } Q_{risk}(s) < \tau \\
\pi_{recovery}(s) & \text{if } Q_{risk}(s) \geq \tau
\end{cases}$$

**训练方式**:
- 任务策略：SAC (Soft Actor-Critic)
- 恢复策略：在危险区域专门训练
- 风险价值：二分类网络

**优点**:
- 简单有效
- 可端到端训练

**缺点**:
- 需要大量碰撞数据训练风险函数
- 零样本泛化能力弱

---

### D.3 PETS (概率集成轨迹采样)

**参考**: Chua et al., NeurIPS 2018

**核心思想**:
使用深度集成 (Deep Ensemble) 估计模型不确定性，通过轨迹采样进行模型预测控制 (MPC)。

**集成模型**:
$$\{f_{\theta_i}\}_{i=1}^M, \quad \theta_i \text{ 独立初始化}$$

**不确定性估计**:
$$\mu(x) = \frac{1}{M}\sum_{i=1}^M f_{\theta_i}(x)$$
$$\sigma^2(x) = \frac{1}{M}\sum_{i=1}^M (f_{\theta_i}(x) - \mu(x))^2$$

**轨迹采样**:
1. 采样初始状态集合 $\{s_0^{(j)}\}$
2. 对每个模型 $f_{\theta_i}$ 和初始状态 $s_0^{(j)}$， rollout H 步
3. 选择期望回报最高的动作序列

**优点**:
- 不确定性估计准确
- 样本效率高

**缺点**:
- 计算开销大 (M 次前向传播)
- 需要调参 (集成大小 M、轨迹长度 H)

---

### D.4 CBF-QP (控制障碍函数二次规划)

**参考**: Ames et al., T-AC 2017; Yuan et al., RA-L 2022

**核心思想**:
使用控制障碍函数 (Control Barrier Function, CBF) 保证系统状态始终在安全集内，通过二次规划 (QP) 求解安全控制输入。

**安全集定义**:
$$\mathcal{C} = \{x \in \mathbb{R}^n : h(x) \geq 0\}$$

**CBF 条件**:
$$\dot{h}(x) + \alpha h(x) \geq 0, \quad \alpha > 0$$

**QP 公式**:
$$\begin{aligned}
\min_{u} \quad & \frac{1}{2}\|u - u_{nom}\|^2 \\
\text{s.t.} \quad & \dot{h}(x) + \alpha h(x) \geq 0 \\
& u \in \mathcal{U}_{adm}
\end{aligned}$$

**优点**:
- 理论保证强 (前向不变集)
- 可处理多约束

**缺点**:
- 高维问题 QP 可能 infeasible
- 需要精确的动力学模型
- 计算耗时 (实时 QP 求解)

---

### D.5 Shielded RL (防护强化学习)

**参考**: Alshiekh et al., ATVA 2018

**核心思想**:
在 RL 策略外层添加运行时防护层 (shield)，拦截危险动作并替换为安全动作。

**防护逻辑**:
```python
def shield(state, action):
    if is_safe(state, action):
        return action  # 动作安全，直接执行
    else:
        return find_safe_action(state)  # 查找最近安全动作
```

**防护实现**:
- 基于模型：使用动力学模型预测动作后果
- 基于规则：手工定义安全规则
- 基于学习：训练安全分类器

**优点**:
- 简单快速
- 可与任何 RL 算法结合

**缺点**:
- 防护逻辑需要手工设计
- 可能过于保守

## 附录 E: 预警提前时间详细分析

### E.1 定义

**预警提前时间** (Warning Lead Time):
$$T_{lead} = T_{violation} - T_{warning}$$

其中:
- $T_{warning}$: RTA 首次触发预警的时间
- $T_{violation}$: 实际物理超包线/碰撞发生时间

**物理解释**: RTA 系统提前多少秒检测到潜在危险并介入。

---

### E.2 各方法预警机制对比

| 方法 | 预警机制 | 提前时间来源 |
|------|----------|--------------|
| **Ours_Full (R2)** | GRU 可达集预测 | 预测未来 1 秒状态，提前 0.5-1.0 秒 |
| **Ours_Full (R3)** | 视觉 OOD 检测 | 感知异常早于动作异常，提前 0.3-0.8 秒 |
| DeepReach | 神经可达集 | 预测未来 1 秒，提前 0.4-0.9 秒 |
| Recovery RL | 风险价值估计 | 基于状态风险，提前 0.2-0.5 秒 |
| PETS | 不确定性估计 | 高不确定性预警，提前 0.3-0.6 秒 |
| CBF-QP | CBF 值监测 | $h(x) < threshold$ 时预警，提前 0.1-0.3 秒 |
| Shielded RL | 规则检查 | 即时检查，提前 0-0.2 秒 |

### E.3 实验统计结果

2. 对每个场景，统计激活最强的神经元
3. 取 Top 6 作为该场景的特征掩码
4. 汉明距离阈值设为 3 (允许 50% 差异)

### G.2 输入输出规格

**底盘 GRU**:
- **输入**: 过去 10 帧底盘状态 $(10, 5)$
  - $[x, y, \theta, v, \omega]$ 每帧
- **输出**: 未来可达包络 8 维支撑函数
  - $[h_0, h_1, ..., h_7]$ (8 个方向)

**机械臂 GRU**:
- **输入**: 过去 10 帧机械臂状态 $(10, 14)$
  - $[q_1...q_7, \dot{q}_1...\dot{q}_7]$ 每帧
- **输出**: 未来可达包络 6 维支撑函数
  - $[h_{x+}, h_{x-}, h_{y+}, h_{y-}, h_{z+}, h_{z-}]$

---

### G.3 支撑向量方向

**底盘 8 方向**:
```
方向索引  角度 (度)  笛卡尔方向
0         0°        +x (前)
1         45°       +x+y (右前)
2         90°       +y (右)
3         135°      -x+y (右后)
4         180°      -x (后)
5         225°      -x-y (左后)
6         270°      -y (左)
7         315°      +x-y (左前)
```

**支撑函数计算**:
$$h_i = \max_{x \in \mathcal{R}} d_i \cdot x$$

其中 $d_i = [\cos(\theta_i), \sin(\theta_i)]$ 是方向向量，$\mathcal{R}$ 是可达集。
