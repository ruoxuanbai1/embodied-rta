## Q1: Region 3 神经掩码针对的模型层数及编号含义

### 1.1 目标 VLA 模型架构

**Region 3 检测的目标模型**: OpenVLA-7B

```
OpenVLA-7B 架构:

│ ├─ RGB 图像 (224×224×3)                                      │
│ └─ 语言指令 (64 tokens)                                     │
└─────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│ Vision Encoder (ViT-B/16)                                   │
│ ├─ Patch Embedding: 14×14=196 patches                       │
│ ├─ 12 个 Transformer Block                                  │
│ │   Block 0: [512 维特征] ← Region 3 检测点 1                │
│ │   Block 1: [512 维特征] ← Region 3 检测点 2                │
│ │   ...                                                     │
│ │   Block 11: [512 维特征] ← Region 3 检测点 12              │
│ └─ 输出：512 维视觉特征向量                                  │
└─────────────────────────────────────────────────────────────┘
         │
         ▼

```

---

### 1.2 神经掩码检测点

**Region 3 检测位置**: Vision Encoder 的输出层 (512 维特征向量)

**为什么选择这个位置**:
1. **语义丰富**: 经过 12 层 ViT 处理，包含高级语义信息
2. **计算高效**: 只需监测 1 个 512 维向量，而非多层
3. **异常敏感**: OOD 输入会导致特征分布显著偏移

### 1.3 掩码编号含义

**8 个掩码模式** (Mask_0 到 Mask_7):

```python
# 掩码库定义 (来自 visual_ood.py)
mask_library = {
    'Mask_0': [5, 12, 23, 45, 67, 89],    # 空旷走廊
    'Mask_1': [8, 15, 28, 42, 55, 78],    # 动态行人
    'Mask_2': [3, 18, 31, 48, 62, 85],    # 低光照
    'Mask_3': [10, 22, 35, 50, 68, 91],   # 抓取任务
    'Mask_4': [7, 19, 33, 46, 61, 84],    # 避障
    'Mask_5': [12, 25, 38, 52, 69, 92],   # 导航
    'Mask_6': [9, 21, 36, 49, 65, 88],    # 高光照
    'Mask_7': [15, 28, 41, 55, 72, 95],   # 复合场景
}
```

**编号含义**:
- **Mask_X**: 第 X 个参考激活模式 (X = 0, 1, ..., 7)
- **列表中的数字**: 512 维特征向量中**激活最强的神经元索引**

**示例解释**:
```
Mask_0: [5, 12, 23, 45, 67, 89]
        ↑   ↑   ↑   ↑   ↑   ↑
        │   │   │   │   │   └─ 第 89 号神经元在"空旷走廊"场景下通常激活
        │   │   │   │ └───── 第 67 号神经元在"空旷走廊"场景下通常激活
        │   │   │   └─────── 第 45 号神经元在"空旷走廊"场景下通常激活
        │   │   └─────────── 第 23 号神经元在"空旷走廊"场景下通常激活
        │   └─────────────── 第 12 号神经元在"空旷走廊"场景下通常激活
        └─────────────────── 第 5 号神经元在"空旷走廊"场景下通常激活
```

**工作原理**:
1. 提取当前 512 维特征向量
2. 找出激活最强的 Top 6 神经元索引
3. 与 8 个参考掩码对比汉明距离
4. 如最小距离 ≥ 3，判定为异常

**可视化**:
```
正常"空旷走廊"场景:
特征向量：[0.1, 0.3, 0.8, ..., 2.1, ..., 1.5, ..., 0.2]
           ↑5        ↑12              ↑67      ↑89
Top 6 索引：[5, 12, 23, 45, 67, 89]
匹配 Mask_0: 汉明距离 = 0 ✅ 正常

异常场景 (对抗攻击):
特征向量：[1.8, 0.2, 0.1, ..., 0.3, ..., 2.5, ..., 1.9]
           ↑0        ↑3               ↑70      ↑95
Top 6 索引：[0, 3, 15, 38, 70, 95]
匹配 Mask_0: 汉明距离 = 6 ❌ 异常
```

## Q2: 各方法的提前预警机制详解

### 2.1 预警机制对比表

| 方法 | 预警类型 | 检测对象 | 提前时间 | 预警公式 |
|------|----------|----------|----------|----------|
| **Ours (R2)** | 预测式 | 可达集边界 | 0.5-1.0s | $\hat{x}_{t+H} \notin \mathcal{R}_{safe}$ |
| **Ours (R3)** | 感知式 | 视觉特征分布 | 0.3-0.8s | $D_M(f_t, f_{ref}) > 3\sigma$ |

| CBF-QP | 反应式 | CBF 值 | 0.1-0.3s | $h(x) < \epsilon$ |
| Shielded RL | 反应式 | 规则检查 | 0-0.2s | $is\_safe(s,a) = False$ |

def check_region2_warning(obs, gru, env):

**预警逻辑**:
```python
def detect(self, visual_features, policy_output=None):
    # 1. OOD 检测 (马氏距离)
    is_ood, ood_dist = self.ood_detector.detect(visual_features)
    #    正常：D_M ≈ 1.0
    #    OOD:   D_M > 3.0
    
    # 2. 跳变检测 (时序差分)
    is_jump, delta, deriv = self.jump_detector.detect(visual_features)
    #    正常：Δ < 0.5
    #    跳变：Δ > 0.5 或 dΔ/dt > 0.3
    
    # 3. 熵检测 (输出不确定性)
    is_uncertain, entropy, kl = self.entropy_detector.detect(policy_output)
    #    正常：H < 1.30
    #    不确定：H > 1.30 或 KL > 0.39
    
    # 4. XAI 掩码检测 (激活模式)
    is_abnormal, hamming, mask_id = self.mask_detector.detect(visual_features)
    #    正常：d_H < 3
    #    异常：d_H ≥ 3
    
    # 风险融合
    risk = 0.3*is_ood + 0.25*is_jump + 0.2*is_uncertain + 0.25*is_abnormal
    return risk > 0.4  # 风险阈值
```

时间线:    t=0          t=0.4        t=0.8        t=1.2

**预警逻辑**:
```python
def get_action(self, obs, original_action):
    # 1. 神经网络计算价值函数 V(x,t)
    #    V < 0: 状态在可达集外 (危险)
    #    V ≥ 0: 状态在可达集内 (安全)
    v_value = self.reachability_net(obs['state'])
    
    # 2. 预警判断
    if v_value < 0:
        # 危险：投影到安全边界
        return self.project_to_safe_boundary(original_action)
    
    return original_action
```

#### PETS (概率集成轨迹采样)

```
时间线:    t=0          t=0.3        t=0.6        t=0.9
           │            │            │            │
           │            │            │            ● 轨迹偏离
           │            │            │
           │            ● PETS 计算
           │              集成方差
           │              σ² > τ (预警)
           │
           ● 输入状态

预警提前时间 = 0.9 - 0.3 = 0.6 秒
```

**预警逻辑**:
```python
def get_action(self, obs, original_action):
    # 1. M 个模型独立预测
    predictions = []
    for model in self.ensemble:
        pred = model.predict(obs['state'], original_action)
        predictions.append(pred)
    
    # 2. 计算不确定性 (方差)
    uncertainty = np.var(predictions, axis=0)
    
    # 3. 高不确定性时保守
    if np.mean(uncertainty) > 0.5:
        return self.conservative_action(original_action)
    
    return original_action
```

#### CBF-QP (控制障碍函数)

```
时间线:    t=0          t=0.1        t=0.2        t=0.3
           │            │            │            │
           │            │            │            ● 越界
           │            │            │
           │            │            ● h(x) < ε
           │            │              QP 求解 (预警)
           │            │
           │            ● 计算 CBF 值
           │
           ● 靠近边界

预警提前时间 = 0.3 - 0.2 = 0.1 秒
```

**预警逻辑**:
```python
def get_action(self, obs, original_action):
    # 1. 计算 CBF 值
    h = self.distance_to_boundary(obs['state']) - self.safety_margin
    
    # 2. CBF 条件检查
    if h < 0:
        # 已越界：紧急停止
        return self.emergency_stop()
    elif h < 0.5:
        # 接近边界：QP 求解安全动作
        return self.solve_cbf_qp(original_action, h)
    
    return original_action
```

```
时间线:    t=0          t=0.05       t=0.1        t=0.15
           │            │            │            │
           │            │            │            ● 危险动作

           │            │
           │            ● 动作输出
           │

```

**预警逻辑**:
```python
def shield(state, action):
    # 1. 规则检查
    if action['v'] > 1.0:  # 速度超限
        return {'v': 1.0, 'ω': action['ω']}
    
    if distance_to_obstacle(state) < 0.5:  # 太近
        return {'v': 0.0, 'ω': 0.0}  # 停止
    
    return action  # 动作安全，直接执行
```

### 2.3 预警机制分类

**预测式预警** (Proactive):
- Ours (R2): GRU 预测未来状态
- DeepReach: 神经可达集
- PETS: 不确定性估计

**特点**: 提前时间长 (0.5-1.0s)，但需要准确模型

**感知式预警** (Perceptive):
- Ours (R3): 视觉特征异常

**特点**: 提前时间中等 (0.3-0.8s)，检测感知层异常

**反应式预警** (Reactive):
- Recovery RL: 风险价值
- CBF-QP: CBF 值监测
- Shielded RL: 规则检查

**特点**: 提前时间短 (0-0.3s)，但响应快速可靠

## Q4: 被测对象任务说明

### 4.1 任务定义

**任务类型**: 视觉语言导航 + 移动操作 (Vision-Language Navigation + Manipulation)

**任务流程**:
```
1. 接收自然语言指令
   "去厨房拿一瓶水" / "导航到目标点并避障"

2. 视觉感知
   ├─ RGB 图像输入 (224×224×3)
   ├─ 深度图像输入 (224×224×1)
   └─ 提取 512 维视觉特征

3. 语言理解
   ├─ 指令编码 (64 tokens)
   └─ 与视觉特征融合

4. 动作生成
   ├─ 底盘速度 [v, ω]
   └─ 机械臂扭矩 [τ1-7]

5. 执行与反馈
   ├─ 执行动作
   ├─ 接收新观测
   └─ 循环至任务完成
```

---

### 4.2 具体任务场景

**本实验中的任务**: **定点导航 + 动态避障**

```
任务描述:
- 起点：(0, 0)
- 目标点：(10m, 0)
- 成功判据：到达目标点 (误差 <0.5m) 且无碰撞

环境挑战:
- 静态障碍物：墙壁、家具
- 动态障碍物：行人、推车
- 感知干扰：光照突变、摄像头遮挡、对抗补丁
- 动力学干扰：负载变化、关节摩擦
```

**改进方案**:
```python
# 真实 FGSM 对抗攻击
import torch
from torchvision import transforms

def generate_adversarial_patch(model, image, target_class):
    """生成对抗补丁"""
    image.requires_grad = True
    output = model(image)
    loss = nn.CrossEntropyLoss()(output, target_class)
    loss.backward()
    
    # FGSM 攻击
    epsilon = 0.15
    adversarial_image = image + epsilon * image.grad.sign()
    adversarial_image = torch.clamp(adversarial_image, 0, 1)
    
    return adversarial_image.detach()
```
